# CMS + Storefront — built for AI agents

A self-hosted CMS that grew an e-commerce flow and a first-party MCP server. Admins compose pages out of registry-driven section blocks; AI agents (Claude Code, Cursor, Continue) drive the same surfaces through a typed, scoped tool API.

![Admin shell](https://funisimo.pro/screenshots/admin-shell.png)

## What's in the box

- **Content management** — 17 section module types (Hero / ProjectCard / Timeline / SkillPills / Services / Testimonials / Manifesto / List / Gallery / Carousel / RichText / PlainText / PlainImage / BlogFeed / SocialLinks / StatsCard / ProjectGrid). Drag-and-drop reorder, per-module style enums, inline translation editing.
- **Themes** — file-backed presets seeded into Mongo, live preview, CSS-token cascade, "reset to preset" overwrite.
- **Internationalization** — `next-i18next` + a CSV translation editor with side-by-side compare; merge-on-save preserves untouched keys; disk + Mongo kept in sync.
- **Publishing** — versioned snapshots with per-snapshot audit trail and one-click rollback.
- **Blog** — Posts collection, drafts, blog feed module.
- **E-commerce**:
  - **Customer auth** — separate `kind: 'customer'` from admins; Credentials + Google providers; cookie-scoped sessions; account dashboard, profile, addresses.
  - **Products** — full CRUD service mirroring Posts shape; admin pane; public `/products` + `/products/[slug]` with ISR; warehouse-synced products marked read-only on the admin form.
  - **Cart** — guest carts in Redis (signed cookie, 30-day sliding TTL); customer carts in Mongo; merge on sign-in; price snapshot at add-time, hard re-validation at checkout.
  - **Inventory / warehouse** — pluggable adapter interface; ships with `MockAdapter` (dev/test) and `GenericFeedAdapter` (any HTTPS URL returning JSON / NDJSON / CSV / TSV with bearer/apiKey/basic auth, four pagination styles, ETag/Last-Modified caching). Per-field manual override locks.
  - **Checkout** — multi-step Amazon-style flow (cart → address → shipping → payment → review → confirmation); `Orders` collection with state machine; mock payment provider with Stripe slot-in; idempotent authorize + finalize; 30-min draft sweeper.
- **MCP server** — first-party stdio MCP server exposing 19 tools across 8 domains (`page.*`, `module.*`, `i18n.*`, `theme.*`, `product.*`, `inventory.*`, `site.*`, `audit.*`). Bcrypt-hashed scoped tokens; every call audited as `actor: 'mcp:<token-name>'`.
- **SEO** — `getStaticProps` + `getStaticPaths` per page → real static HTML; ISR (`revalidate: 60`) refreshes on admin edits; `sitemap.xml` + `robots.txt` are generated by `next-sitemap` post-build; per-page `<title>` / `<meta>` / OpenGraph driven by the `SiteSeoService` admin form; `next-i18next` `hreflang` tags on every locale variant.
- **Auth** — NextAuth Credentials + Google for both admin and customer populations, JWT sessions carrying `kind` + `role`, bcrypt hashing, IP rate-limiting, progressive sign-in lockout with live countdown, first-boot admin password generated to a protected artefact file.
- **Method-level authz** — central `guardMethods` Proxy gates every service method by role (admin/editor/viewer) or principal (customer/guest), and stamps `_session.email` / `_session.customerId` on injected methods for audit.
- **Tests** — Vitest + `mongodb-memory-server` (293 unit/integration tests, all green); Playwright e2e suite under `tests/e2e/` with per-worker Mongo + Next dev (Phase 1 specs landed; full module coverage rolls out in phases).
- **Linux deployment** — `infra/compose.yaml` for prod + `infra/compose.dev.yaml` for local Mongo. `Scripts/deploy.sh` rsyncs the build to a DigitalOcean droplet.

## Stack

- Next.js 16 (pages router, Turbopack in dev), React 19, TypeScript 5
- Ant Design v6 + custom SCSS + reveal-on-scroll animations
- GraphQL via Apollo Server (Next API route) + standalone Express server option
- MongoDB 7 — `Navigation`, `Sections`, `Users`, `Languages`, `Themes`, `SiteSettings`, `PublishedSnapshots`, `Posts`, `Images`, `Logos`, `Products`, `Carts`, `Orders`, `StockReservations`, `Counters`, `InventoryRuns`, `InventoryDeadLetters`, `McpTokens`, `Audit`
- Redis 7 (cart guest store)
- NextAuth (Credentials + Google for admins and customers); bcrypt hashing; progressive lockout
- next-i18next + CSV translation editor
- `@modelcontextprotocol/sdk` for the MCP server
- `@playwright/test` for e2e

## Quickstart

See [docs/SETUP.md](docs/SETUP.md) for the full first-time setup walkthrough (Linux + Windows + macOS), including how to register the local MCP server with Claude Code or Cursor.

Short version:

```bash
yarn --frozen-lockfile          # or npm install --legacy-peer-deps
cp .env.example .env.local      # fill in NEXTAUTH_SECRET, MONGODB_URI, ADMIN_DEFAULT_PASSWORD
docker compose -f infra/compose.dev.yaml up -d   # Mongo on 27017
docker run -d -p 6379:6379 redis:7-alpine        # Redis for the cart
npm run dev                     # http://localhost/admin
```

Sign in: `admin@admin.com` / whatever you set as `ADMIN_DEFAULT_PASSWORD`.

## MCP server (built for AI agents)

The CMS ships its own MCP server. Claude Code / Cursor / Continue connect to it over stdio and get a typed tool catalogue:

| Domain | Tools |
|---|---|
| Pages / sections | `page.list`, `page.get`, `page.create`, `section.update`, `section.delete` |
| Modules | `module.listTypes` |
| Translations | `i18n.listLanguages`, `i18n.upsertKeys` |
| Themes | `theme.list`, `theme.update`, `theme.setActive` |
| Products | `product.list`, `product.get`, `product.create`, `product.update`, `product.publish` |
| Inventory | `inventory.status`, `inventory.syncDelta`, `inventory.readDeadLetters` |
| Site | `site.revalidate` |
| Audit | `audit.list` |

Tokens are issued from the admin UI (`Settings → MCP`) with scoped permissions (`read:content`, `write:products`, …). Every tool call is audited as `actor: 'mcp:<token-name>'`.

To register with Claude Code, copy [tools/mcp.example.json](tools/mcp.example.json), paste your token, and drop it into your `~/.claude/mcp.json`.

## Scripts

| Script | Purpose |
|---|---|
| `npm run dev` | Next dev on port 80 (Turbopack); auto-runs `dev:db` (Docker Mongo) |
| `npm run dev:db` / `dev:db:stop` | Bring the local Mongo container up / down |
| `npm run standalone-graphql` | Standalone GraphQL server (port 80) |
| `npm run build` / `npm start` | Production build / serve `.next/` |
| `npm run mcp:stdio` | Boot the MCP server over stdio (consumed by Claude Code / Cursor) |
| `npm run mcp:http` | Stub for the future HTTP/SSE MCP transport |
| `npm test` / `:watch` / `:coverage` | Vitest |
| `npm run e2e` / `:headed` / `:install` | Playwright |
| `./Scripts/deploy.sh` | Local build + rsync + remote PM2 reload |
